Step 3: Supabase Dashboard — Authentication

  Go to Supabase > Your Project > Authentication > URL Configuration:

  - Site URL: Set to https://collaboly.vercel.app
  - Redirect URLs: Add https://collaboly.vercel.app/api/auth/callback
  - Remove any http://localhost:3000 entries (or keep if you still do local dev against
   this project)

  Go to Authentication > Providers > Email:

  - Confirm email: Make sure this is OFF — your app expects immediate sign-in after
  signup
  - Minimum password length: Set to 8 (matches your Zod passwordSchema)
  - Max password length: Leave default (72)

  Go to Authentication > Rate Limits:

  - Verify defaults are reasonable (typically 4 emails/hour, ~100 signups/hour). Don't
  change unless you have a reason.

  ---
  Step 4: Supabase Dashboard — Database

  Go to Supabase > Settings > Database > Connection string:

  - Make sure you're using the Pooler URL in Transaction mode (not Session mode). It
  should look like: postgresql://postgres.xxxxx:PASSWORD@aws-X-us-east-1.pooler.supabas
  e.com:6543/postgres
  - This is what you put in DATABASE_URL in Step 2

  ---
  Step 5: Run database migrations

  From your local machine, with DATABASE_URL pointing to your production Supabase
  database:

  npx drizzle-kit migrate

  This applies all migrations in drizzle/ (including the new 0031 cascade delete
  migration).

  ---
  Step 6: Redeploy

  Trigger a new deploy on Vercel (push a commit or click "Redeploy" in the dashboard)
  so it picks up the environment variables.

  ---
  Step 7: Test on the live Vercel URL

  1. Sign up — create a new account, verify you land on /about
  2. Sign in — log out and back in
  3. Password reset — request a reset, check email (and spam), click link, set new
  password
  4. Delete account — create a throwaway account and delete it from Profile
  5. Check headers — open DevTools > Network > click any request > Response Headers.
  Verify X-Frame-Options: DENY, X-Content-Type-Options: nosniff,
  Strict-Transport-Security are present

  2. Vercel: Change NEXT_PUBLIC_SITE_URL from https://collaboly.vercel.app to

  3. Supabase: Update both the Site URL and Redirect URL from collaboly.vercel.app to

  Then re-test the password reset flow since it's the only thing affected by the URL
